# Contributing to the Companion Repository

Thank you for taking the time to improve this repository! Contributions from readers make the companion materials better for everyone.

## Ways to Contribute

### Report Errors or Broken Links
If you find a typo, an incorrect code example, a broken link, or outdated information:

1. Open an **Issue** on GitHub with the label `bug` or `broken-link`.
2. Include: the file path, the line number (if applicable), and a brief description of the problem.

### Suggest Improvements
Have a better code example, an additional exercise, or a clarifying note?

1. Open an **Issue** with the label `enhancement` describing your suggestion.
2. For small fixes, open a **Pull Request** directly.

### Submit a Pull Request

1. **Fork** this repository.
2. Create a feature branch: `git checkout -b fix/chapter-03-typo`
3. Make your changes. Keep each PR focused on a single fix or improvement.
4. Commit with a clear message: `git commit -m "Fix: incorrect output in chapter-03 main.py"`
5. Push and open a Pull Request against the `main` branch.

## Code Style Guidelines

- Python: follow [PEP 8](https://pep8.org/). Run `flake8` before submitting.
- JavaScript/TypeScript: use `eslint` with the project's config.
- All languages: keep examples self-contained and runnable without additional setup.

## What NOT to Contribute

- Changes that contradict the book's content without citing a source.
- Large refactors that change the instructional purpose of an example.
- Dependencies that are not clearly needed.

## Questions?

Reach out at **support@JambaAcademy.com** — we respond to all questions about the book and companion materials.
